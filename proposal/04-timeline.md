## Phased Delivery Plan

Total engagement length: **12 weeks**

Milestones are cumulative — each phase builds on the previous. No feature work begins before the architecture review is complete.

---

### Phase 1 — Onboarding & Discovery (Weeks 1–2)

| Deliverable                                                                      | Week |
|----------------------------------------------------------------------------------|------|
| R4: Architecture review — codebase walkthrough, component map, data flow diagram | 1    |
| R4: Architecture documentation published (HTML, written narrative)               | 2    |
| R1: Reports module audit — defect classification and findings document           | 2    |

**Milestone:** Architecture document and Reports defect list delivered to Meridian IT for review.

---

### Phase 2 — Reports Remediation (Weeks 3–5)

| Deliverable                                                                | Week |
|----------------------------------------------------------------------------|------|
| R1: Filter wiring defects resolved                                         | 3    |
| R1: i18n gaps resolved                                                     | 4    |
| R1: Data pattern inconsistencies resolved                                  | 4    |
| R3: Playwright tests for Reports flows committed                           | 5    |
| R1: Remediation complete — findings document updated with resolution notes | 5    |

**Milestone:** Reports module fully remediated, tests passing, available for Meridian IT review.

---

### Phase 3 — Restocking View (Weeks 6–9)

| Deliverable                                                            | Week |
|------------------------------------------------------------------------|------|
| R2: Backend API endpoint(s) designed and reviewed with Meridian        | 6    |
| R2: Backend implementation complete                                    | 7    |
| R2: Frontend Restocking view — recommendation display and budget input | 8    |
| R2: Frontend — accept/adjust/dismiss interactions complete             | 9    |
| R3: Playwright tests for Restocking flow committed                     | 9    |

**Milestone:** Restocking view feature-complete, tests passing, available for Meridian IT review.

---

### Phase 4 — Hardening (Weeks 10–12)

Reserved for:
- Scope that surfaces during the Reports audit beyond the initial defect list
- Feedback and revision cycles from Meridian IT on Phases 2 and 3
- Performance, edge-case, and cross-warehouse testing
- Final documentation updates and handoff preparation

**Milestone (Week 12):** Engagement complete. All deliverables accepted, test suite handed to Meridian IT, final architecture documentation updated to reflect delivered state.
