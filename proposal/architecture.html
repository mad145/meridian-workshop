Known Technical Debt

Items inherited from the previous vendor. Relevant to R1 remediation scope.

Critical

Reports.vue bypasses the API wrapper

Calls axios directly with hardcoded URLs. Filters (warehouse, category, period) have no effect on Reports data. Root cause of the filter wiring defects in R1.

Critical

Reports.vue uses Options API

All other views use Composition API. Reports was left in the older pattern. Inconsistency makes it harder to test and maintain. Should be refactored to match the rest of the codebase.

Moderate

5 unimplemented backend endpoints declared in api.js

Tasks and purchase-order endpoints are called from the frontend but return 404. Frontend works around this with mock data in useAuth.js.

Moderate

Manual number formatting in Reports.vue

Custom formatNumber() duplicates logic already in utils/currency.js. Formatting is locale-unaware — will produce incorrect output for Japanese locale.

Low

Dashboard.vue also uses Options API

Options API migration was started but not completed. Dashboard is a lower priority than Reports for this engagement but is a long-term maintainability concern.

Low

purchase_orders.json is an empty array

The purchase order data file exists but contains no data. The PurchaseOrderModal is referenced in Dashboard but appears incomplete.

Stack Summary

Frontend

  • Framework Vue 3 (Composition API)
  • Build Vite, port 3000
  • HTTP Axios (centralised in api.js)
  • Routing vue-router
  • i18n Custom composable (EN, JP)
  • Charts Custom SVG — no library
  • Styles Scoped CSS, custom design tokens
  • TypeScript No — plain JavaScript

Backend

  • Framework Python FastAPI, port 8001
  • Runtime Uvicorn (via uv)
  • Data JSON files, in-memory at runtime
  • Models Pydantic v2
  • Filtering In-memory list comprehensions
  • CORS allow_origins=["*"] (dev only)
  • Database None
  • Tests None delivered
